34 /*
35  @namespace Tangram
36  The Tangram namespace houses all of the code within Tangram.
37 
38  Cells (aka zones/elements) are the highest dimension entities in a mesh
39  Nodes (aka vertices) are lowest dimension entities in a mesh
40  Faces in a 3D mesh are 2D entities, in a 2D mesh are 1D entities
41  BOUNDARY_FACE is a special type of entity that is need so that process
42  kernels can define composite vectors (see src/data_structures) on
43  exterior boundary faces of the mesh only
44 
45  Wedges are special subcell entities that are a simplicial
46  decomposition of cell. In 3D, a wedge is tetrahedron formed by one
47  point of the edge, the midpoint of the edge, the "center" of the
48  face and the "center" of the cell volume. In 2D, a wedge is a
49  triangle formed by an end-point of the edge, the mid-point of the
50  edge and the center of the cell. In 1D, wedges are lines, that are
51  formed by the endpoint of the cell and the midpoint of the
52  cell. There are two wedges associated with an edge of cell face in
53  3D. The two wedges associated with an edge and face pair form a larger
54  simplex called a side
55 
56  Corners are also subcell entities that are associated uniquely with
57  a node of a cell. Each corner is the union of all the wedges incident
58  upon that node in the cell
59 
60  Facets are the boundary entity between two wedges in adjacent
61  cells. In 3D, a facet is a triangular subface of the cell face
62  shared by two wedges in adjacent cells. In 2D, a facet is half of
63  an edge that is shared by two wedges in adjacent cells
64  */

212 /* In the context of interface reconstruction methods based on the nested
213  dissections algorithm, the iterative methods are used to find the
214  position of the cutting plane.
215  VOF uses an iterative method to find the cutting distance for a given normal,
216  in which case the objective function is volume and fun_eps is volume tolerance,
217  while arg_eps is unused.
218  MOF, in addition to that, iteratively solves for the normal. In this case,
219  the tolerances are used not for the objective function, but the argument,
220  and arg_eps will be imposed on the change in polar angles between consecutive
221  steps. Hence, MOF can use one instance of IterativeMethodTolerances_t with
222  fun_eps used for finding the cutting distance, and arg_eps used for finding
223  the direction of the normal.
224 */
225 struct IterativeMethodTolerances_t {
226  int max_num_iter; // Max number of iterations
227  double arg_eps; // Tolerance on the arguments of the function
228  double fun_eps; // Tolerance on the value of the function
229 };
